### **Semantic Memory in LangGraph**
```
Semantic memory = stored knowledge or facts the agent knows over time.

It is not conversation history and not past executions.

👉 It is structured knowledge the agent can recall anytime.
```

*Examples:*
```
User name = Rahul
Preferred language = Python
Company policy rules
Product information
Documentation knowledge

These are facts, not events.
```

#### ⭐**What Semantic Memory Stores**
```
Semantic memory keeps stable information:
    User preferences
    Important facts
    Learned information
    Domain knowledge
    Long-term context
```

#### ⭐**How LangGraph Uses Semantic Memory**
```
Usually implemented using:
    Vector databases
    Key-value stores
    Knowledge graphs
    External databases
```
*Flow:*
```
User gives information
        ↓
Extract fact
        ↓
Store in semantic memory
        ↓
Retrieve when needed
```

### 🔴***Difference Between Semantic Memory and Factual Memory in LangGraph***
```
Factual Memory	=>  Individual stored facts
Semantic Memory	=>  Organized knowledge system built from facts
```
```
👉 Factual memory = data
👉 Semantic memory = understanding built from data
```
